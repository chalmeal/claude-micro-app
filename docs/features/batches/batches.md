## 概要

管理者がシステム内のバッチジョブを管理する機能。スケジュール設定・有効/無効切り替え・手動再実行・実行履歴確認・実行ログ閲覧ができる。管理者専用機能。

---

## ユーザーストーリー

```
管理者として、
バッチジョブの実行状況を把握するために、
一覧で各バッチのステータス・最終実行日時・次回実行予定を確認したい。

管理者として、
バッチの実行タイミングを変更するために、
スケジュール（頻度・時刻・曜日/日）を設定したい。

管理者として、
特定のバッチを一時停止・再開するために、
有効/無効を切り替えたい。

管理者として、
失敗したバッチを手動で再実行するために、
即時再実行をトリガーしたい。

管理者として、
バッチの失敗原因を調査するために、
実行ログを確認したい。
```

---

## 機能要件

### 必須（Must）

- [x] バッチ一覧を確認できる（ステータス・最終実行・次回実行表示）
- [x] バッチ詳細と実行履歴を確認できる（ページネーション付き）
- [x] スケジュールを更新できる（分毎・時毎・日毎・週毎・月毎）
- [x] 有効/無効を切り替えられる
- [x] 即時再実行できる
- [x] 実行ログを確認できる（info / warn / error レベル）

### 対象外（Won't）

- 新規バッチジョブの作成（コード変更が必要なため）
- バッチジョブの削除

---

## 受け入れ条件

- [x] 管理者以外のアクセスは403エラー
- [x] スケジュール変更後に `nextRunAt` が再計算される
- [x] 実行履歴は新しい順に表示、ページネーション（最大100件/ページ）
- [x] 実行ログはレベル（info/warn/error）・タイムスタンプ・メッセージを表示

---

## 画面・遷移の概要

- バッチ管理画面（`/admin/batches`）: 一覧・スケジュール編集・実行履歴・ログ確認を1画面で管理
