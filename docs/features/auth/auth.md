## 概要

メールアドレスとパスワードによるJWT認証を提供する。ログイン・パスワード変更・パスワードリセット（メール経由）・初回パスワード設定の4つのフローを含む。

---

## ユーザーストーリー

```
ユーザーとして、
メールアドレスとパスワードでログインするために、
ログイン画面から認証してJWTトークンを取得したい。

ユーザーとして、
パスワードを忘れた場合に自力で再設定できるために、
メール経由でパスワードリセットリンクを受け取りたい。

管理者として、
新規ユーザーに安全にアカウントを開設させるために、
初期パスワード設定用のリンクをメールで送りたい。
```

---

## 機能要件

### 必須（Must）

- [x] メール・パスワードでログインし、JWTトークンを取得できる
- [x] ログイン中にパスワードを変更できる（現在のパスワード確認必須）
- [x] パスワードを忘れた場合にリセット用メールを受け取れる
- [x] メール内のトークンで新しいパスワードを設定できる（有効期限: 1時間）
- [x] 管理者が新規ユーザーを作成した際、初期パスワード設定用メールが送られる（有効期限: 7日）
- [x] 非アクティブ（inactive）ユーザーはログインできない

### 対象外（Won't）

- OAuth / ソーシャルログイン
- 二段階認証
- セッション管理（ステートレスJWT）

---

## 受け入れ条件

- [x] 正しいメール・パスワードでログインするとJWTトークンが返る
- [x] 誤ったパスワードでは401エラーが返る
- [x] `inactive` ユーザーはログインできず401エラーが返る
- [x] パスワードリセットメールはメールアドレスが存在しない場合も200を返す（ユーザー列挙攻撃対策）
- [x] 期限切れのリセットトークンは400エラーとなる
- [x] 新規パスワードは8文字以上でなければならない

---

## 画面・遷移の概要

- ログイン画面 → ダッシュボード（認証成功時）
- ログイン画面 → パスワード忘却画面
- パスワード忘却画面 → ログイン画面（メール送信後）
- パスワードリセット画面（メールのリンク）→ ログイン画面
- 初回パスワード設定画面（メールのリンク）→ ログイン画面

## 備考・決定事項

- パスワードはbcryptでハッシュ化して保存
- リセットトークンは `users` テーブルの `passwordResetToken` / `passwordResetExpiresAt` カラムで管理
- パスワードリセットとアカウント初期設定でトークンの仕組みを共用（有効期限が異なる）
