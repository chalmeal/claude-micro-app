## 概要

管理者操作の証跡を自動記録し、管理者が参照できる機能。ユーザー作成・更新、バッチ操作、お知らせ管理など主要な操作が対象。参照のみで、ログの作成・削除は内部処理でのみ行われる。

---

## ユーザーストーリー

```
管理者として、
誰がいつ何の操作をしたかを把握するために、
監査ログをフィルタリングして一覧で確認したい。
```

---

## 機能要件

### 必須（Must）

- [x] 監査ログの一覧を確認できる（管理者のみ）
- [x] ユーザーID・アクション・日時範囲でフィルタリングできる
- [x] ページネーション（最大100件/ページ）に対応している
- [x] 主要な操作が自動的にログ記録される

### 記録される操作

| アクション | 説明 |
| --- | --- |
| `auth.login` | ログイン |
| `auth.change_password` | パスワード変更 |
| `user.create` | ユーザー作成 |
| `user.update` | ユーザー更新 |
| `batch.schedule_update` | バッチスケジュール更新 |
| `batch.toggle_enabled` | バッチ有効/無効切り替え |
| `batch.rerun` | バッチ再実行 |
| `announcement.create` | お知らせ作成 |
| `announcement.update` | お知らせ更新 |
| `announcement.delete` | お知らせ削除 |

### 対象外（Won't）

- ログの手動作成・編集・削除（UIから操作不可）
- 成績操作のログ記録

---

## 受け入れ条件

- [x] 管理者以外のアクセスは403エラー
- [x] 全フィルタパラメータはオプション（指定なしで全件表示）
- [x] 日時は新しい順に表示される
