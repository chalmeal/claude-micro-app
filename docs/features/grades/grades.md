## 概要

学生の科目ごとの成績（スコア・評価）を管理する機能。スコアを登録すると評価（S〜F）が自動計算される。認証済みユーザー全員が参照・登録・編集・削除できる。

---

## ユーザーストーリー

```
ユーザーとして、
学生の成績を記録するために、
学生ID・科目・スコアを入力して成績を登録したい。

ユーザーとして、
成績の傾向を把握するために、
評価・年度・学期・キーワードで絞り込んで一覧を確認したい。

ユーザーとして、
入力ミスを修正するために、
登録済みの成績のスコアを編集したい。
```

---

## 機能要件

### 必須（Must）

- [x] 成績を登録できる（スコア入力で評価を自動計算）
- [x] 成績一覧を取得できる（評価・年度・学期・キーワードでフィルタ）
- [x] 成績詳細を確認できる
- [x] 成績のスコアを更新できる（評価も再計算される）
- [x] 成績を削除できる
- [x] 同一学生ID・科目・年度・学期の重複登録を防ぐ

### あると望ましい（Should）

- [x] CSV形式での一括登録ができる（フロントエンド側でCSVをパースし、1件ずつPOSTする）

### 対象外（Won't）

- 統計・グラフ表示

---

## 受け入れ条件

- [x] スコア0〜100の範囲外は400エラー
- [x] 同一学生ID・科目・年度・学期の組み合わせが重複すると409エラー
- [x] スコア更新時に評価が自動再計算される
- [x] 一覧は30件ずつページネーション表示
- [x] 検索状態はsessionStorageに保持され、画面遷移後に復元される

---

## スコア→評価換算ルール

| スコア | 評価 |
| --- | --- |
| 90〜100 | S |
| 80〜89 | A |
| 70〜79 | B |
| 60〜69 | C |
| 50〜59 | D |
| 0〜49 | F |

---

## 画面・遷移の概要

- 一覧画面（`/grades`）: フィルタフォーム + 一覧テーブル + ページネーション
- 詳細画面（`/grades/:id`）: 成績情報表示・削除ボタン
- 登録画面（`/grades/new`）: 登録フォーム
- 編集画面（`/grades/:id/edit`）: スコア編集フォーム
