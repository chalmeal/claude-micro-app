# 画面設計書: 初回パスワード設定画面

| 項目 | 内容 |
| --- | --- |
| 作成日 | 2026-05-13 |
| 更新日 | 2026-05-13 |
| ステータス | 完了 |
| パス | `/setup-password?token=<token>` |
| 認証 | 不要 |

---

## 概要

管理者がユーザーを新規作成した際に送信されるメールのリンクから遷移する画面。初めてのログイン前にパスワードを設定する。リセットパスワード画面と同じUIだが、トークンの有効期限が7日と長い。

---

## 画面構成（ワイヤーフレーム）

```
+----------------------------------------+
|   パスワードの設定                      |
|   （初回ログイン用）                    |
|                                        |
|   新しいパスワード（8文字以上）        |
|   [______________________________]     |
|                                        |
|   [エラーメッセージ（ある場合）]       |
|                                        |
|       [パスワードを設定する]           |
+----------------------------------------+
```

---

## コンポーネント一覧

| コンポーネント | 説明 |
| --- | --- |
| パスワード入力 | type="password", required, minlength=8 |
| エラーメッセージ | トークン無効・期限切れ等のエラー表示 |
| 設定ボタン | submit, 送信中は無効化 |

---

## 状態と表示

| 状態 | 表示内容 |
| --- | --- |
| 初期表示 | フォームのみ |
| 送信中 | ボタンをローディング表示・無効化 |
| トークン無効/期限切れ（7日） | エラーメッセージを表示（管理者への再招待依頼を促す） |
| 成功 | ログイン画面へ遷移 |

---

## ユーザー操作と挙動

| 操作 | 挙動 |
| --- | --- |
| フォーム送信 | URLのtokenとpasswordをPOST → 成功時はログイン画面へ遷移 |

---

## 画面遷移

```
初回パスワード設定画面（/setup-password?token=...）
  ├─ 設定成功    → ログイン画面（/login）
  └─ トークン無効 → エラー表示（同一画面）
```

---

## 使用するAPI

| メソッド | エンドポイント | 用途 |
| --- | --- | --- |
| POST | `/api/auth/reset-password/confirm` | パスワード確定（リセットと共用） |

---

## 備考・決定事項

- `/reset-password` と同じAPIエンドポイント・ロジックを共用している
- トークンの有効期限のみ異なる（リセット: 1時間、初回設定: 7日）
