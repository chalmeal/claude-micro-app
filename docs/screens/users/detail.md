# 画面設計書: ユーザー詳細画面

| 項目 | 内容 |
| --- | --- |
| 作成日 | 2026-05-13 |
| 更新日 | 2026-05-13 |
| ステータス | 完了 |
| パス | `/users/:id` |
| 認証 | 管理者のみ |

---

## 概要

特定ユーザーの情報を確認し、ロールとステータスを変更できる管理者向け画面。

---

## 画面構成（ワイヤーフレーム）

```
+------------------------------------------+
| ← ユーザー一覧に戻る                      |
|                                          |
| ユーザー詳細                              |
|                                          |
| 名前:        山田太郎                    |
| メール:      yamada@example.com          |
| 作成日:      2026-01-01                  |
|                                          |
| ロール:      [admin ▼]                  |
| ステータス:  [active ▼]                 |
|                                          |
| [エラーメッセージ（ある場合）]           |
|                                          |
|              [保存する]                  |
+------------------------------------------+
```

---

## コンポーネント一覧

| コンポーネント | 説明 |
| --- | --- |
| ユーザー情報表示 | 名前・メール・作成日（読み取り専用） |
| ロール選択 | `admin` / `member` のセレクトボックス |
| ステータス選択 | `active` / `inactive` のセレクトボックス |
| 保存ボタン | PATCH API を呼び出す |
| 一覧へ戻るリンク | `/users` へ遷移 |

---

## 状態と表示

| 状態 | 表示内容 |
| --- | --- |
| 読み込み中 | スピナー |
| 表示中 | ユーザー情報とフォーム |
| 保存中 | ボタンをローディング表示・無効化 |
| 保存成功 | スナックバーで「更新しました」を表示 |
| エラー | エラーメッセージを表示 |

---

## ユーザー操作と挙動

| 操作 | 挙動 |
| --- | --- |
| 保存ボタンクリック | PATCH `/api/users/:id` → 成功時にスナックバー表示 |
| 一覧へ戻るリンク | `/users` へ遷移 |

---

## 画面遷移

```
ユーザー詳細（/users/:id）
  └─ 一覧へ戻る → ユーザー一覧（/users）
```

---

## 使用するAPI

| メソッド | エンドポイント | 用途 |
| --- | --- | --- |
| GET | `/api/users/:id` | ユーザー情報取得 |
| PATCH | `/api/users/:id` | ロール・ステータス更新 |
