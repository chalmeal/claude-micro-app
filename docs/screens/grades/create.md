## 概要

新しい成績レコードを登録する画面。1件登録・まとめて登録・CSV取り込みの3モードを持つ。studentId はフロントエンドが自動生成（既存成績の studentName から同一人物を検索し、なければ UUID を採番）するため、ユーザーが入力する必要はない。スコアを入力すると評価はサーバー側で自動計算される。

---

## 画面構成（ワイヤーフレーム）

```
+------------------------------------------+
| ← 成績一覧に戻る                          |
|                                          |
| 成績登録                                  |
|                                          |
| [1件登録] [まとめて登録] [CSV取り込み]    |
| ─────────────────────────                |
|                                          |
| ── 1件登録 ──                           |
| 年度 *                                   |
| [2024 ▼]                                 |
|                                          |
| 学期 *                                   |
| (●) 前期（spring）  ( ) 後期（fall）     |
|                                          |
| 学生名 *                                 |
| [______________________________]         |
|                                          |
| 科目 *                                   |
| [数学I ▼]（15科目から選択）              |
|                                          |
| スコア（0〜100） *                       |
| [______]  → 評価: [A]（リアルタイム表示）|
|                                          |
|              [登録する]                  |
|                                          |
| ── まとめて登録 ──                      |
| 科目: [▼]  年度: [▼]  学期: [▼]（共通設定）|
|                                          |
| +-----------+-------+                   |
| | 学生名    | スコア |                   |
| +-----------+-------+                   |
| | [_______] | [___] |                   |
| | [_______] | [___] |                   |
| +-----------+-------+                   |
| [+ 行を追加]（空の行は無視されます）     |
|              [登録する]                  |
|                                          |
| ── CSV取り込み ──                       |

```

---

## コンポーネント一覧

| コンポーネント | 説明 |
| --- | --- |
| タブ切り替え | 1件登録 / まとめて登録 / CSV取り込みの3モード |
| 1件登録フォーム | year・semester・studentName・subject・score 入力。score変更時に評価をリアルタイム表示 |
| まとめて登録フォーム | 科目・年度・学期を共通設定し、学生名・スコアを行ごとに入力 |
| CSV取り込みフォーム | CSVファイル選択・解析後プレビュー表示・エラー行の案内 |
| 確認ダイアログ | 登録内容（学生名・科目・スコア・評価・年度・学期）を確認してから実行 |

---

## 1件登録フォームのフィールド

| フィールド | 型 | 必須 | バリデーション | 初期値 |
| --- | --- | --- | --- | --- |
| year | number | 必須 | 2023 or 2024 の選択肢 | 2024 |
| semester | 'spring' \| 'fall' | 必須 | — | 'spring' |
| studentName | string | 必須 | 空白でない | '' |
| subject | string | 必須 | SUBJECTS一覧から選択 | SUBJECTS[0] |
| score | number | 必須 | 整数 0〜100 | 0 |

---

## まとめて登録フォームのフィールド

**共通設定（全行に適用）**

| フィールド | 型 | 初期値 |
| --- | --- | --- |
| subject | string（15科目から選択） | SUBJECTS[0] |
| year | number | 2024 |
| semester | 'spring' \| 'fall' | 'spring' |

**行ごとのフィールド**

| フィールド | 型 | 説明 |
| --- | --- | --- |
| studentName | string | 空白の行は送信時に除外 |
| score | number | 整数 0〜100 |

---

## 科目一覧（SUBJECTS）

数学I / 数学II / 英語 / 国語 / 物理 / 化学 / 生物 / 歴史 / 経済学 / プログラミング入門 / データ構造 / アルゴリズム論 / 線形代数 / 統計学 / 情報倫理

---

## CSV取り込みフォーマット

```csv
studentName,subject,score,year,semester
田中太郎,数学I,85,2024,spring
鈴木花子,英語,72,2024,fall
```

| カラム | 必須 | 許容値 |
| --- | --- | --- |
| studentName | 必須 | 任意の文字列（空は除外） |
| subject | 必須 | 任意の文字列（空は除外） |
| score | 必須 | 整数 0〜100 |
| year | 必須 | 整数 2000〜2100 |
| semester | 必須 | `spring` または `fall` |

バリデーションエラーの行はスキップされ、有効な行のみ登録される。

---

## 状態と表示

| 状態 | 表示内容 |
| --- | --- |
| 初期表示 | 1件登録フォーム（タブ選択中） |
| CSV解析後 | プレビューテーブルとエラーメッセージ（エラーあれば）を表示 |
| 確認ダイアログ | 登録内容のサマリーを表示 |
| 送信中 | ボタンをローディング表示・無効化 |
| 登録成功 | スナックバー「登録しました」 → 一覧へ遷移 |
| 重複エラー | `N行目: 同じ学生・科目・年度・学期の成績が既に存在します` を表示 |

---

## ユーザー操作と挙動

| 操作 | 挙動 |
| --- | --- |
| 1件登録フォーム送信 | 確認ダイアログ → POST `/api/grades` → 一覧へ遷移 |
| まとめて登録フォーム送信 | 確認ダイアログ → POST `/api/grades` を有効行ごとに順次実行 → 一覧へ遷移 |
| CSVアップロード | フロントエンドでCSVをパース → プレビュー表示 |
| CSV登録ボタン | 確認ダイアログ → POST `/api/grades` を各行に対して順次実行 → 一覧へ遷移 |
| 一覧へ戻るリンク | `/grades` へ遷移 |

---

成績登録（/grades/new）

## 使用するAPI

| メソッド | エンドポイント | 用途 |
| --- | --- | --- |
| POST | `/api/grades` | 成績登録（モードに関わらず1件ずつ呼び出し） |

---

## 備考・決定事項

- studentId はフロントエンドが既存成績の studentName から逆引きし、未登録なら UUID を生成して送信する（ユーザー入力不要）
- まとめて登録・CSV取り込みともに、専用の一括登録APIはなく POST `/api/grades` を件数分順次呼び出す
- 途中でエラーが発生した場合、エラー行より前の登録は完了済みとなる（ロールバックなし）
