---

## 概要

既存のお知らせを編集する管理者向け画面。既存データをフォームの初期値として表示し、保存する。

---

## 画面構成（ワイヤーフレーム）

```
+------------------------------------------+
| ← お知らせ管理に戻る                      |
|                                          |
| お知らせ編集                              |
|                                          |
| タイトル *                               |
| [システムメンテナンスのお知らせ__________] |
|                                          |
| 本文 *                                   |
| [5月20日 2:00〜4:00 にメンテナンス...]   |
|                                          |
| 日付 *                                   |
| [2026-05-15__________]                   |
|                                          |
| カテゴリ *                               |
| ( ) 重要  ( ) お知らせ  (●) メンテナンス |
|                                          |
| [エラーメッセージ（ある場合）]           |
|                                          |
|              [保存する]                  |
+------------------------------------------+
```

---

## コンポーネント一覧

新規作成画面と同じコンポーネント構成。初期値として既存データが入力済みの状態で表示される。

---

## 状態と表示

| 状態 | 表示内容 |
| --- | --- |
| 読み込み中 | スピナー |
| 表示中 | 既存データが入力済みのフォーム |
| 保存中 | ボタンをローディング表示・無効化 |
| 保存成功 | 管理一覧へ遷移 |
| エラー | エラーメッセージを表示 |

---

## ユーザー操作と挙動

| 操作 | 挙動 |
| --- | --- |
| フォーム送信 | PUT `/api/announcements/:id` → 成功時に管理一覧へ遷移 |
| 管理一覧へ戻るリンク | `/admin/announcements` へ遷移 |

---

## 画面遷移

```
お知らせ編集（/admin/announcements/:id/edit）
  ├─ 保存成功      → お知らせ管理一覧（/admin/announcements）
  └─ 一覧へ戻る   → お知らせ管理一覧（/admin/announcements）
```

---

## 使用するAPI

| メソッド | エンドポイント | 用途 |
| --- | --- | --- |
| GET | `/api/announcements/:id` | 既存データ取得（フォーム初期値） |
| PUT | `/api/announcements/:id` | お知らせ更新 |
